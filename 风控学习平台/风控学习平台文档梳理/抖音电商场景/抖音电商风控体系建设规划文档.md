## 一、规划概述

### 1.1 规划背景

抖音电商以"内容+货架"双阵地为核心架构，用户全流程覆盖"进入平台-浏览种草-详情查看-加购-提单-支付-履约发货-收货确认"九大环节。作为风控产品经理，需要构建覆盖用户全旅程的风控体系，识别各业务节点的潜在风险，设计合理的处置策略和申诉机制，保障平台交易安全与用户体验。

### 1.2 规划目标

#### 1.2.1 业务目标

| 目标维度 | 具体指标 | 目标值 | 完成状态 |
|---------|---------|--------|---------|
| **风险覆盖** | 风险场景覆盖率 | 100% | ⬜ 待完成 |
| **识别能力** | 风险识别准确率 | >95% | ⬜ 待完成 |
| **处置效率** | 风险处置响应时效 | <100ms | ⬜ 待完成 |
| **用户体验** | 误拦截率 | <0.5% | ⬜ 待完成 |
| **申诉处理** | 申诉处理时效 | <4小时 | ⬜ 待完成 |

#### 1.2.2 规划范围

覆盖用户全流程的九大业务节点：

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│进入平台 │ → │浏览商品 │ → │查看详情 │ → │加入购物车│ → │提交订单 │ → │支付结算 │ → │物流履约 │ → │确认收货 │ → │售后评价 │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
    │              │              │              │              │              │              │              │              │
 账号风险      流量/内容风险    交易准备风险    库存占用风险    交易欺诈风险    支付安全风险    物流欺诈风险    资金结算风险    售后套利风险
```

### 1.3 规划原则

1. **风险导向**：优先识别和处置高风险场景
2. **分级管理**：建立风险分级体系，差异化处置
3. **用户优先**：平衡安全与体验，减少误伤
4. **闭环管理**：识别-处置-申诉-优化的完整闭环
5. **持续迭代**：基于数据反馈持续优化策略

## 二、分场景风控体系规划

### ✅ 场景一：注册登录阶段

> **状态**：已完成  
> **完成日期**：2026-02-08  
> **相关文档**：
> - [注册登录业务流程说明文档](./注册登录阶段/注册登录业务流程说明文档.md)

#### 2.1 风险识别思路

**2.1.1 需要识别的风险类型**

| 风险类别 | 风险行为 | 识别维度 | 识别思路 |
|---------|---------|---------|---------|
| **虚假注册** | 批量机器注册 | 设备维度 | 同一设备短时间内注册多个账号；设备信息异常或缺失 |
| | 虚拟号码注册 | 号码维度 | 使用虚拟运营商号段；号码活跃度低 |
| | 接码平台注册 | 行为维度 | 号码历史注册多个平台；注册后无正常使用行为 |
| **账号盗用** | 异常登录 | 环境维度 | 登录设备/地点/时间与历史习惯差异大 |
| | 暴力破解 | 行为维度 | 短时间内多次登录失败；使用常见密码组合 |
| | 撞库攻击 | 数据维度 | 使用其他平台泄露的账号密码尝试登录 |
| **设备伪造** | 模拟器注册 | 设备维度 | 设备特征不符合真实设备；传感器数据缺失 |
| | 参数篡改 | 设备维度 | 设备参数逻辑矛盾；存在Root/越狱痕迹 |

**2.1.2 风险识别数据需求**

| 数据类型 | 数据内容 | 用途 |
|---------|---------|------|
| 设备数据 | 设备型号、系统版本、设备指纹 | 识别设备异常、设备聚集 |
| 行为数据 | 注册时间、操作路径、停留时长 | 识别机器行为、异常模式 |
| 号码数据 | 运营商、归属地、历史行为 | 识别虚拟号码、接码平台 |
| 环境数据 | IP地址、地理位置、网络类型 | 识别环境异常、位置聚集 |
| 历史数据 | 账号历史行为、设备历史记录 | 识别账号盗用、行为突变 |

#### 2.2 处置策略设计

**2.2.1 风险分级体系**

| 风险等级 | 判定标准 | 处置措施 | 响应时效 |
|---------|---------|---------|---------|
| **极高风险** | 确认黑产攻击、批量恶意注册 | 直接拦截；记录黑名单 | 实时 |
| **高风险** | 高度疑似风险行为 | 强制二次验证（短信+人脸）；限制功能 | 实时 |
| **中风险** | 存在一定风险特征 | 滑块验证码；加强监控 | 实时 |
| **低风险** | 正常用户行为 | 正常通过；记录日志 | 实时 |

**2.2.2 处置流程设计**

```
用户操作 → 风险识别 → 风险评分 → 等级判定 → 策略执行 → 结果反馈
    │           │           │           │           │           │
    ▼           ▼           ▼           ▼           ▼           ▼
注册/登录    多维度分析    综合评分     匹配等级     执行处置     告知用户
```

**2.2.3 特殊场景处置**

| 场景 | 处置策略 | 说明 |
|-----|---------|------|
| 新设备登录 | 二次验证 | 首次在新设备登录需验证身份 |
| 异地登录 | 提醒+验证 | 异地登录时发送提醒，异常时需验证 |
| 频繁注册 | 限制+审核 | 同一设备/号码注册频次超限后限制 |

#### 2.3 申诉机制设计

**2.3.1 申诉场景**

| 申诉类型 | 触发条件 | 申诉方式 |
|---------|---------|---------|
| **注册被拦截** | 正常用户被误判为虚假注册 | 自助申诉+人工审核 |
| **登录受限** | 正常用户触发风控被限制登录 | 身份验证+人工复核 |
| **功能限制** | 正常用户被限制部分功能 | 在线申诉+客服介入 |

**2.3.2 申诉流程**

```
用户发起申诉 → 系统自动核验 → 人工审核（如需） → 结果反馈 → 补偿（如误判）
      │               │               │               │               │
      ▼               ▼               ▼               ▼               ▼
  提交材料      备用验证方式      专员核实      通知用户结果     发放补偿
```

**2.3.3 申诉处理时效**

| 处理环节 | 时效要求 | 处理标准 |
|---------|---------|---------|
| 自助申诉 | <5分钟 | 基于备用验证方式自动核验 |
| 人工审核 | <4小时 | 综合评估用户历史行为、申诉材料 |
| 复核申请 | <24小时 | 高级专员二次审核 |

**2.3.4 误判补偿机制**

| 误判类型 | 补偿措施 | 执行标准 |
|---------|---------|---------|
| 注册被误拦截 | 优先审核通道+优惠券补偿 | 确认误判后即时处理 |
| 登录被误限 | 立即解除限制+服务补偿 | 确认误判后即时处理 |
| 功能被误限 | 解除限制+积分/优惠券补偿 | 确认误判后4小时内处理 |

#### 3.1 风险识别思路

**3.1.1 需要识别的风险类型**

| 风险类别 | 风险行为 | 识别维度 | 识别思路 |
|---------|---------|---------|---------|
| **流量作弊** | 机器刷量 | 行为维度 | 观看时长异常短；无真实交互行为 |
| | 协议刷量 | 技术维度 | 直接调用API；无正常渲染行为 |
| | 众包刷量 | 聚集维度 | 多设备地理位置聚集；行为时间同步 |
| **内容违规** | 虚假宣传 | 内容维度 | 功效夸大描述；价格误导信息 |
| | 违禁品推广 | 内容维度 | 销售禁售商品；擦边球描述 |
| | 导流站外 | 内容维度 | 评论区/私信留联系方式；图片隐含水印 |
| **恶意引流** | 黑灰产引流 | 关联维度 | 批量账号发布相似内容；引导至第三方 |
| | 水军控评 | 聚集维度 | 批量账号集中评论；内容高度相似 |

**3.1.2 风险识别数据需求**

| 数据类型 | 数据内容 | 用途 |
|---------|---------|------|
| 行为数据 | 观看时长、滑动轨迹、点击行为 | 识别刷量行为、机器特征 |
| 内容数据 | 文本、图片、视频内容 | 识别违规内容、虚假宣传 |
| 账号数据 | 账号历史行为、关联关系 | 识别水军团伙、黑产网络 |
| 设备数据 | 设备分布、聚集情况 | 识别众包刷量、设备农场 |

#### 3.2 处置策略设计

**3.2.1 风险分级体系**

| 风险等级 | 判定标准 | 处置措施 | 响应时效 |
|---------|---------|---------|---------|
| **极高风险** | 确认违禁品、严重违规 | 立即下架内容；封禁账号 | <10分钟 |
| **高风险** | 明显违规/作弊行为 | 限制内容曝光；要求整改 | <30分钟 |
| **中风险** | 疑似违规/作弊 | 降权处理；增加审核频率 | <2小时 |
| **低风险** | 正常内容 | 正常展示；抽样审核 | 实时 |

**3.2.2 处置流程设计**

```
内容发布/行为发生 → 风险识别 → 内容审核 → 处置执行 → 结果通知
        │               │           │           │           │
        ▼               ▼           ▼           ▼           ▼
    用户上传内容    多维度分析    人工/AI审核    执行策略    通知用户
```

**3.2.3 商家/创作者处置**

| 处置措施 | 适用场景 | 执行标准 |
|---------|---------|---------|
| 内容下架 | 违规内容 | 确认违规后立即下架 |
| 流量降权 | 疑似作弊 | 限制内容推荐曝光 |
| 功能限制 | 多次违规 | 限制发布、评论等功能 |
| 账号封禁 | 严重违规 | 永久封禁，清退处理 |

#### 3.3 申诉机制设计

**3.3.1 申诉场景**

| 申诉类型 | 触发条件 | 申诉方式 |
|---------|---------|---------|
| **内容被下架** | 创作者认为内容被误判违规 | 在线申诉+材料提交 |
| **流量被降权** | 创作者认为被误判作弊 | 申诉+数据说明 |
| **账号被处罚** | 商家/创作者认为处罚不当 | 人工申诉+举证 |

**3.3.2 申诉处理时效**

| 处理环节 | 时效要求 | 处理标准 |
|---------|---------|---------|
| 申诉受理 | <1小时 | 确认收到申诉，告知处理流程 |
| 审核处理 | <24小时 | 审核内容、评估申诉理由 |
| 结果反馈 | 审核后<2小时 | 告知审核结果及理由 |

**3.3.3 误判补偿**

| 误判类型 | 补偿措施 |
|---------|---------|
| 内容被误下架 | 恢复内容+流量补偿+书面道歉 |
| 流量被误降权 | 恢复权重+流量券补偿 |
| 账号被误处罚 | 撤销处罚+信用修复+补偿方案 |

| **优惠套现** | 优惠券套现 | 关联维度 | 优惠券使用异常；关联订单异常 |
| | 满减套利 | 行为维度 | 凑单后退单；价格异常波动 |
| | 新人福利套利 | 聚集维度 | 多账号领取新人福利；设备关联 |

**4.1.2 风险识别数据需求**

| 数据类型 | 数据内容 | 用途 |
|---------|---------|------|
| 订单数据 | 订单金额、商品信息、下单时间 | 识别异常订单、套利模式 |
